# Jumping to Chapter Books — 프로젝트 규칙

## 서비스 개요
- 서비스명: Jumping to Chapter Books
- 대상: 빛이 (초등 5학년)
- 목적: 리더스북 자막으로 단어/해석/퀴즈 학습 사이트 자동 생성

## 폴더 구조 규칙
루트/
  index.html         ← 메인 허브 (시리즈 선택)
  arthur/
    index.html       ← 아서 시리즈 목록
    arthur-*.html    ← 권별 학습 페이지
  henry-mudge/
    index.html       ← 헨리앤머지 시리즈 목록
    henry-mudge-*.html
  [새시리즈]/        ← 추가 시 동일 구조 적용

## 디자인 규칙 (모든 파일 공통)

### 폰트
- 한국어: Noto Sans KR (Google Fonts)
- 영어: Nunito (Google Fonts)
- 절대 궁서체, 명조체 사용 금지

### 시리즈별 대표 색상 (추가 시 목록에 등록)
- 아서 시리즈: #FF8C42 (주황)
- 헨리앤머지 시리즈: #2EC4B6 (청록)
- 새 시리즈 추가 시 기존과 겹치지 않는 색상 배정

### 색상 적용 원칙
- 메인 index.html 시리즈 카드 색상
- 시리즈 index.html 헤더 색상
- 권별 학습 페이지 포인트 색상
- 위 세 곳 모두 동일한 시리즈 대표색 사용

## 네비게이션 규칙
- 모든 시리즈 index.html → "← 시리즈 홈" 버튼 필수
- 모든 권별 학습 페이지 → "← 시리즈 목록으로" 버튼 필수
- 상단 "Jumping to Chapter Books" 클릭 → 루트 index.html 이동
- 뒤로가기 버튼 없는 페이지 절대 만들지 말 것

## 학습 페이지 기능 규칙
모든 권별 학습 페이지는 아래 3탭 구조 필수:
- Words 탭: 단어카드 + TTS 발음 + 검색
- Story 탭: 영한 나란히, 한국어 기본 숨김, 문장별 토글 + 전체 토글
- Quiz 탭: 영어 객관식 10문제 + 점수 + 재시도

## 반응형 규칙
- 768px 이하: 카드 1열, 폰트 축소
- 터치 버튼 최소 44px
- 모든 파일 모바일 대응 필수

## 권수 관리 규칙
- 시리즈 index.html의 전체/완성/준비중 숫자는 실제 파일 기준
- 새 권 HTML 파일 생성 시 → 시리즈 index.html 자동 업데이트
- 새 권 생성 시 → 루트 index.html 완성 권수도 자동 업데이트
- 준비중 카드 클릭 시 토스트 메시지: "곧 업데이트 예정이에요 😊"

## 새 시리즈 추가 규칙
새 시리즈 추가 요청 시 아래를 자동으로 처리:
1. 시리즈 폴더 생성
2. 시리즈 index.html 생성 (기존 시리즈와 동일 구조)
3. 루트 index.html에 시리즈 카드 추가
4. 이 CLAUDE.md 시리즈 색상 목록에 추가
5. 대표 색상 세 곳 모두 통일 적용

## 새 권 추가 규칙
새 권 추가 요청 시 아래를 자동으로 처리:
1. 권별 학습 페이지 HTML 생성
2. 시리즈 index.html 완성 권수 업데이트
3. 루트 index.html 완성 권수 업데이트
4. 위 디자인/네비게이션/반응형 규칙 자동 적용

## 배포 환경
- GitHub Pages 배포 예정
- 외부 API 의존 없이 순수 HTML/CSS/JS
- TTS는 Web Speech API 사용 (브라우저 내장)

## 번역 원칙 (모든 권에 동일 적용)

### 단어 카드 (words 배열 ko 필드)
- 사전적 원래 의미를 먼저 적는다
- 필요 시 어근·품사 변화를 괄호 안에 표기한다
- 형식: `"한국어 뜻 (어근 또는 문법 설명)"`

### 본문 번역 (story 배열 ko 필드)
- 직역 원칙: 영어 구조를 한국어로 최대한 그대로 따라간다
- 원문에 없는 의미를 추가하지 않는다
- 문법 패턴이 드러나도록 번역한다
- 약간 딱딱하더라도 직역을 유지한다 — 아이가 문맥으로 의미를 유추하고 확장하는 것이 목적

## 구글 드라이브 업로드 (권별 자동화)
새 권 생성 시 아래 두 파일을 구글 드라이브에 자동 업로드:
1. 단어 정리 → 구글 시트 (번호/영어/한국어/난이도 4열)
2. 원문 & 해석 → 구글 문서 (EN+KO 나란히, 장면별 구분)
- 파일명 형식: "Arthur [제목] — 단어 정리", "Arthur [제목] — 원문 & 해석"
- 업로드 방식: Google Drive API 또는 MCP 연동
